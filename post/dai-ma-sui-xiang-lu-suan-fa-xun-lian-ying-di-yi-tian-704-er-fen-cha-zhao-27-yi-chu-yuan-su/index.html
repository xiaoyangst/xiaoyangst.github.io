704. 二分查找

题目地址：https://leetcode.cn/problems/binary-search/

个人题解：没有参考他人代码完成

class Solution {
public:
    int search(vector<int>& nums, int target) {
        int left = 0;
        int right = nums.size() - 1;
        int middle = (right + left)/2;
        for(int i = left; i <= middle+1; i++){
            if(target == nums[middle]){
                return middle;
            }else if(target > nums[middle]){
                left = middle + 1;
                middle = (right + left)/2;
            }else{
                right = middle - 1;
                middle = (right+ left)/2;
            }
        }
        return -1;
    }
};

评价：哪怕曾经做过这道题，时隔多月，却忘得是一干二净。通过不断提交提示用例错误来不断调整代码，最终完成用例的提交。凭着记忆，我知道这里面middle的计算方式可以被优化来防止数值越界问题，而且这个值在if判断之后记得重新计算，因为每次左右边界不同之后，中间值也就跟着变化。还有关于left或right值得变化可以用图示的方式轻易得出，不要弄错。还有一个地方是for循环语句的第二个参数，这地方容易出错，要知道是i <= middle+1。

该数组为有序数组（升序），且无重复元素（如果有，下标将不唯一）。这两个前提条件是使用二分法的前提条件

老师让我们用两种方式去写二分法：

  • 定义 target 是在一个在左闭右闭的区间里，也就是[left, right]
  • 定义 target 是在一个在左闭右开的区间里，也就是[left, right)

不同的区间表达式，处理的方法也是截然不同的，左闭右闭的区间前面已经演示，但是需要进行优化，所以下面会讲这两种方式的代码。

左闭右闭的区间里：

class Solution {
public:
    int search(vector<int>& nums, int target) {
        int left = 0;
        int right = nums.size() - 1; // 定义target在左闭右闭的区间里，[left, right]
        while (left <= right) { // 当left==right，区间[left, right]依然有效，所以用 <=
            int middle = left + ((right - left) / 2);// 防止溢出 等同于(left + right)/2
            if (nums[middle] > target) {
                right = middle - 1; // target 在左区间，所以[left, middle - 1]
            } else if (nums[middle] < target) {
                left = middle + 1; // target 在右区间，所以[middle + 1, right]
            } else { // nums[middle] == target
                return middle; // 数组中找到目标值，直接返回下标
            }
        }
        // 未找到目标值
        return -1;
    }
};

评价：

  1. 前面我讲要注意for循环中记住第二个参数要知道是i <= middle+1，但是在while循环中就可以忽视这个问题，而且while循环判断条件的不同，处理的区间方式也就不同，比方说这里的条件是left<=right，而下面的左闭右开的区间的条件是left<right。
  2. middle的计算：left + ((right - left) / 2)，可以防止越界。如果right和left很大的话，就会出现越界问题，而如果先通过(right - left) / 2进行计算再和left相加的话，可以相对的避免越界问题。还有代码的优化部分是middle计算的位置，我之前多出存在同样的相同代码，尽管能够通过，但是代码的冗余可以降低，也是值得一提的。
  3. 还有一点是一开始right的计算，要知道如果nums中有6个元素，那么 nums.size()计算的结果也是6，由于我们最后返回下标，所以right = nums.size() - 1。

左闭右开的区间里：

class Solution {
public:
    int search(vector<int>& nums, int target) {
        int left = 0;
        int right = nums.size(); // 定义target在左闭右开的区间里，即：[left, right)
        while (left < right) { // 因为left == right的时候，在[left, right)是无效的空间，所以使用 <
            int middle = left + ((right - left) >> 1);
            if (nums[middle] > target) {
                right = middle; // target 在左区间，在[left, middle)中
            } else if (nums[middle] < target) {
                left = middle + 1; // target 在右区间，在[middle + 1, right)中
            } else { // nums[middle] == target
                return middle; // 数组中找到目标值，直接返回下标
            }
        }
        // 未找到目标值
        return -1;
    }
};

评价：

  1. while (left < right)，这里使用 < ,因为left == right在区间[left, right)是没有意义的
  2. 正因为右边是开区间，所以在right = middle

27. 移除元素

题目地址：https://leetcode.cn/problems/remove-element/

个人题解：无，没有做出来

双指针法

class Solution {
public:
    int removeElement(vector<int>& nums, int val) {
            //快慢指针一开始指向同一个下标
            int slowIndex = 0;
            int fastIndex = 0;
            //快指针开始移动
            for(fastIndex; fastIndex < nums.size(); fastIndex++){
                if(nums[fastIndex] != val){
                    nums[slowIndex] = nums[fastIndex];
                    slowIndex++;
                }
            }
            return slowIndex;
    }
};

不管是暴力破解还是双指针法，我都没有能够成功，阅读老师的代码之后有些疑惑，但是经过思考之后就明白其中的美妙了，下面详细谈一谈：

  1. 通过一个快指针和慢指针在一个for循环下完成两个for循环的工作。因为快指针和慢指针指向数组的起始位置，对于这两个指针而言，通过它们就等同于拥有这个数组，因为数组中数据的排列是线性的，只不过指向的是同一个数组。快指针：寻找新数组的元素 ，新数组就是不含有目标元素的数组；慢指针：指向更新 新数组下标的位置。
  2. 题意说明只需要把非目标元素往前存储，能够通过新数组长度遍历出来即可，无需在意元素的顺序。所以，在快指针遍历元素判断为非目标元素之后，把当前快指针所指向的元素覆盖慢指针当前所指向的元素即可，由于操作的是同一个数组，在无需创造新数组的情况下就能够解决该题。

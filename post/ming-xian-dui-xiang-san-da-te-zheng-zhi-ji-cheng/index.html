面向对象三大特征之继承

继承的目的

实际上，在封装篇已经涉及继承，当时为了探究三种访问权限的意义，不得以引入继承的相关内容，而本篇章目的是继续学习继承的内容，关于继承的目的如下：

  1. 代码重用： 继承允许子类继承父类的属性和行为，使得子类可以重用父类的代码。子类可以通过继承父类的成员变量和成员函数，无需重新编写相同的代码，提高了代码的复用性。
  2. 扩展功能： 继承允许子类在父类的基础上添加新的成员变量和成员函数，以扩展父类的功能。子类可以添加额外的属性和行为，使得子类具有父类的特性并增加自己的特有功能，实现了代码的扩展性。
  3. 多态性支持： 继承是实现多态性的基础。通过将对象引用指向父类，可以在运行时使用不同的子类对象，实现多态行为。多态性允许以统一的方式处理不同类型的对象，提高了代码的灵活性和可扩展性。
  4. 抽象和分类： 继承关系可以帮助我们将对象进行分类和组织。通过将共同属性和行为放在父类中，可以实现对对象的抽象和分类，提高代码的可维护性和可理解性。

总的来说，继承的目的是为了实现代码的重用和扩展，通过定义一个通用的父类和派生出不同的子类，可以更好地组织和管理代码，提高代码的可维护性和可扩展性。

继承的本质就是复制

权限继承

在继承中，成员变量和成员函数的访问权限不仅和父类自身的访问权限有关，同时也和子类在继承父类时设置的权限继承有关。权限继承修改的对象是派生类（子类），而非基类（父类）

这里面有规律可循：min(父类访问权限，子类继承方式权限)

public继承基类：不改变基类成员的访问控制

private继承基类：所有成员都无法被访问，全为private

protected继承基类：基类中的private成员的访问控制不变，其余的都变为protected

简单来说，普遍使用public继承基类，因为基类成员的权限是怎么样的就是怎么样，就不用过多考虑基础后子类的权限情况。

子类继承父类的继承方式格式：

class 子类名 : 访问权限修饰符 父类名 {
    // 子类的成员变量和成员函数声明
};

我一再强调，继承的本质是复制，那么这种权限继承影响的是谁？绝不是父类，而是子类。

为了讲清楚这个问题，看看下面这份代码

#include <iostream>
#include <string>

using namespace std;

class Student{
public:
    void SpubPri(){
        cout<<"SpubPri() running ..."<<endl;
    }

    int age;

protected:
    void SproPri(){
        cout<<"SproPri() running ..."<<endl;
    }
};

class Test: protected Student{
public:
    int sex;
private:
    string address;
};


int main(){

    Test t;
    //t.age;    /* 必然是失败的 ，因为父类虽然为public，但是Test对象继承权限为protected，无法在外部访问*/

    return 0;
}

t.age当然是不可能成功的，父类虽然为public，但是Test对象继承权限为protected，而protected权限无法在外部访问。

继承父类后的Test成员访问权限情况如下：

public:
    int sex;
protected:
	void SpubPri(){
        cout<<"SpubPri() running ..."<<endl;
    }
    int age;
    void SproPri(){
        cout<<"SproPri() running ..."<<endl;
    }
private:
    string address;

using--修改基类成员在子类中的权限

  1. 在派生类中利用using声明可以改变基类除私有成员外其他成员的访问权限
  2. 改变后该成员在派生类的访问权限由using位于的派生类访问说明符确定
#include <iostream>

using namespace std;

class Student{
public:
    void publicFun(){
        cout<<"Student::publicFun() running..."<<endl;
    }
protected:
    void protectedFun(){
        cout<<"Student::protectedFun() running..."<<endl;
    }
private:
    void privateFun(){
        cout<<"Student::privateFun() running..."<<endl;
    }
};

class Test: private Student{
public:
    using Student::publicFun;
    using Student::protectedFun;
    //using Student::privateFun;    /* 失败，因为using无法修改私有权限 */
    void TestPri(){
        publicFun();
        protectedFun();
    }
};


int main(){

    Test t;
    t.TestPri();

    return 0;
}

输出结果：

Student::publicFun() running...
Student::protectedFun() running...

各位应该清楚，由于权限继承的方式为private，按理无法访问任何父类成员，但是关键字using修饰之后，权限皆为public，也就能够正常访问父类成员。所以，using的作用开始明晰，即对继承过来的父类权限可再进行修改的余地（除父类的私有成员外）

那么，using关键字修饰的父类成员全部变为public权限吗？绝非如此

using修饰之后的父类成员得到的权限取决于你把父类成员放在哪个权限之下，以图片说明：

图中子类权限设置为public，所以被using修饰后权限皆为public。想必你明白如果要把父类继承的成员（除父类的私有成员外）修改为保护权限，只需要放在protected修饰符下即可

using的意义绝不只是非私有权限的修改，还有其他用途，但不属于本篇要讨论的内容

多继承

C++是支持多继承的，开篇就说明，继承的本质就是复制，无外乎多获得另一个类的成员变量和成员函数而已，实际应用中并不推荐这种做法，因为存在命名冲突、二义性、耦合性增加等问题，尽管有一定的策略解决。若非得以，不要出手😂

格式：

class 派生类名 : 访问权限修饰符 基类1, 访问权限修饰符 基类2, ... {
    // 派生类的成员变量和成员函数声明
};

接口继承--纯虚函数

virtual 返回类型 函数名(参数列表) = 0;

一般用于怎样的场景？头文件中用纯虚函数代表此为接口，所以使用纯虚函数的类被称为抽象类，它不能被直接实例化，只能用作接口规范的基类。派生类继承抽象类时，必须实现纯虚函数，以满足接口的要求，这种强制性起到规范的作用。

下面简单看看纯虚函数的声明，以及无法被实例化，必须被子类重写函数才能被调用的模样

#include <iostream>
using namespace std;

// 抽象基类
class Shape {
public:
    virtual double getArea() const = 0;     // 纯虚函数，用作接口规范
    virtual double getPerimeter() const = 0;
};

// 派生类，继承自 Shape 抽象基类
class Circle : public Shape {
private:
    double radius;

public:
    Circle(double r) : radius(r) {}

    double getArea() const {
        return 3.14159 * radius * radius;
    }

    double getPerimeter() const {
        return 2 * 3.14159 * radius;
    }
};

int main() {
    Circle circle(5.0);
    cout << "Area: " << circle.getArea() << endl;
    cout << "Perimeter: " << circle.getPerimeter() << endl;

    return 0;
}

输出结果：

Area: 78.5397
Perimeter: 31.4159
